# 启动说明:

## 一、本地启动步骤:

> 前后端项目部署于本地

### 1.1 启动本地后端：

> 进入 virtual-tryon/backend/

```bash
# 激活已配置的环境
conda activate ootdiffusion

# 启动后端（支持热更新）
uvicorn main:app --reload --port 8000
```

### 1.2 启动本地前端：

> 进入 virtual-tryon/frontend/

```bash
# 安装依赖
npm install

# 启动前端：
npm run dev
```

## 二、远程启动步骤

> 大模型项目部署于远程 autoDL（私有云） 

### 2.1 ssh 隧道端口映射（在本地操作，非远程终端操作）：

> 1.1 通用模板（这个会进入远程终端）：
>
> ```bash
> ssh -p SSH端口 \
>   -L 本机端口:远程目标地址:远程目标端口 \
>   用户名@远程服务器IP
> ```
>
> 1.2 多个端口就写多个 `-L`：
>
> ```bash
> ssh -p SSH端口 \
>   -L 本机端口1:远程目标地址:远程目标端口1 \
>   -L 本机端口2:远程目标地址:远程目标端口2 \
>   用户名@远程服务器IP
> ```
>
> 2. 只做端口转发、不进入远程终端，可以加 `-N`：
>
> ```bash
> ssh -p SSH端口 \
>   -N \
>   -L 本机端口:远程目标地址:远程目标端口 \
>   用户名@远程服务器IP
> ```

例子：

```bash
ssh -p 3485 \
  -L 9000:127.0.0.1:9000 \
  -L 9001:127.0.0.1:9001 \
  root@172.24.246.49
```

### 2.2 启动远程 API 服务（大模型）

> 在远程服务器的 `OOTDiffusion/run/` 目录下执行：

```bash
# 激活已配置的环境（未配置环境，见 ootdiffusion 说明文档）
conda activate ootdiffusion

# 终端 1 — HD 服务（上半身专用，效果最好）
python api_server.py \
  --gpu_id 0 \
  --port 9000 \
  --model_type hd \
  --n_samples 4 \
  --scale 2.0

# 终端 2 — DC 服务（下半身 / 连衣裙）
python api_server.py \
  --gpu_id 0 \
  --port 9001 \
  --model_type dc \
  --n_samples 4 \
  --scale 2.0
```

## <扩展> 脚本启动:

#### Windows 服务器

把整个项目同步到 Windows 服务器,**双击 `start_dev.bat`**。
会自动开 3 个 cmd 窗口:

- Backend(端口 8000)
- Frontend(Vite,通常 5173)
- SSH 隧道(把 GPU 服务器的 9000/9001 映射到本地)

第一次启动会问 SSH 密码 / fingerprint,正常输入即可。

#### Linux GPU 服务器

```bash
cd 项目目录/virtual-tryon/OOTDiffusion/run
bash start_models.sh    # 启动 HD + DC 两个推理服务

# 开启一个终端(跟踪 hd 进度日志):
cd 项目目录/virtual-tryon/OOTDiffusion/run/logs
tail -n 10 -f hd.log  # 观察启动进度

# 开启一个终端(跟踪 dc 进度日志):
cd 项目目录/virtual-tryon/OOTDiffusion/run/logs
tail -n 10 -f dc.log  # 观察启动进度
```

停止:

```bash
# 在原先 start 开启的终端进行 stop
bash stop_models.sh
```
