# 启动说明:

## 一、本地启动步骤:

> 前后端项目部署于本地

### 1.1 启动本地后端：

> 进入 virtual-tryon/backend/

```bash
# 激活已配置的环境
conda activate ootdiffusion

# 启动后端（支持热更新）
uvicorn main:app --reload --port 8000
```

### 1.2 启动本地前端：

> 进入 virtual-tryon/frontend/

```bash
# 安装依赖
npm install

# 启动前端：
npm run dev
```

## 二、远程启动步骤

> 大模型项目部署于远程 autoDL（私有云） 

### 2.1 ssh 隧道端口映射（在本地操作，非远程终端操作）：

> 1.1 通用模板（这个会进入远程终端）：
>
> ```bash
> ssh -p SSH端口 \
>   -L 本机端口:远程目标地址:远程目标端口 \
>   用户名@远程服务器IP
> ```
>
> 1.2 多个端口就写多个 `-L`：
>
> ```bash
> ssh -p SSH端口 \
>   -L 本机端口1:远程目标地址:远程目标端口1 \
>   -L 本机端口2:远程目标地址:远程目标端口2 \
>   用户名@远程服务器IP
> ```
>
> 2. 只做端口转发、不进入远程终端，可以加 `-N`：
>
> ```bash
> ssh -p SSH端口 \
>   -N \
>   -L 本机端口:远程目标地址:远程目标端口 \
>   用户名@远程服务器IP
> ```

例子：

```bash
ssh -p 3485 \
  -L 9000:127.0.0.1:9000 \
  -L 9001:127.0.0.1:9001 \
  root@172.24.246.49
```

### 2.2 启动远程 API 服务（大模型）

> 在远程服务器的 `OOTDiffusion/run/` 目录下执行：

```bash
# 激活已配置的环境（未配置环境，见 ootdiffusion 说明文档）
conda activate ootdiffusion

# 终端 1 — HD 服务（上半身专用，效果最好）
python api_server.py \
  --gpu_id 0 \
  --port 9000 \
  --model_type hd \
  --n_samples 4 \
  --scale 2.0

# 终端 2 — DC 服务（下半身 / 连衣裙）
python api_server.py \
  --gpu_id 0 \
  --port 9001 \
  --model_type dc \
  --n_samples 4 \
  --scale 2.0
```
